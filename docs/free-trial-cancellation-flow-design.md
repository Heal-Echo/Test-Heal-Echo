# 무료 체험 해지 플로우 설계 문서

> 작성일: 2026-03-19
> 목적: `/mypage/settings/subscription/cancel` 페이지의 고객 여정 설계 근거
> 선행 문서: `free-trial-user-types.md`, `similar-service-trial-day1-3-strategy.md`

---

## 1. 힐에코 프로젝트 성격과 해지 맥락

### 1-1. 서비스 특성

힐에코는 영상 기반 웰니스 솔루션 서비스입니다. 핵심 가치는 "하루 15분, 꾸준한 실천으로 자율신경 균형 회복"이며, 고객이 즉각적 효과보다는 점진적 변화를 체감하는 구조입니다. 이 특성은 해지 플로우 설계에 결정적 영향을 줍니다. 7일 체험 기간은 효과를 충분히 체감하기에 짧고, 그래서 "아직 경험하지 못한 가치"를 전달하는 것이 핵심 리텐션 전략이 됩니다.

### 1-2. 결제 구조

토스페이먼츠 빌링키 방식입니다. 무료 체험 시작 시 카드가 등록되고, 7일 후 자동 결제가 시작됩니다. 해지하면 `billing-cancel` Lambda가 PaymentsTable의 빌링키 상태를 `cancelled`로, SubscriptionsTable의 구독 유형을 `free_stopped`으로 변경합니다. 해지 후에도 체험 종료일까지 콘텐츠 이용이 가능합니다.

### 1-3. 해지를 요청하는 고객 유형

`free-trial-user-types.md`에 정의된 5가지 유형 중 해지를 실행하는 유형은 다음과 같습니다:

| 유형 | 해지 시점 | 해지 동기 | 리텐션 가능성 |
|---|---|---|---|
| B (충동적 등록자) | 6~7일차 | 결제 임박 불안 | 중간 — 콘텐츠 재참여 유도 시 |
| C (비교 쇼핑객) | 3~5일차 | 다른 서비스 선택 | 낮음 — 차별화 어필 시 |
| D (해지 예정자) | 5~6일차 | 처음부터 해지 계획 | 매우 낮음 |
| E (불안한 결제자) | 0~1일차 | 불안 해소 실패 | 높음 — 안심 정보 제공 시 |

유형 A(진지한 탐색자)는 해지하지 않으므로 이 플로우의 대상이 아닙니다.

## 2. 유사 서비스 해지 플로우 비교 분석

### 2-1. Headspace (명상/웰니스)

Headspace의 해지 플로우는 업계에서 가장 세련된 사례로 꼽힙니다.

**플로우 단계:**
1. "구독 취소" 클릭
2. 해지 사유 선택 (5~6개 선택지)
3. 사유에 따른 맞춤 리텐션 오퍼 제시 — "비용" 선택 시 50% 할인, "시간 부족" 선택 시 일시 정지 제안
4. "잃게 되는 것" 화면 — 진행 중인 프로그램, 연속 명상일, 수면 기록 등 개인화된 데이터 강조
5. 최종 확인 — "정말 취소하시겠습니까?"
6. 해지 완료 + 언제든 다시 시작할 수 있다는 메시지

**핵심 전략:** 사유 기반 맞춤 리텐션. 모든 고객에게 같은 오퍼를 제시하지 않고, 이탈 원인에 맞는 해결책을 제안합니다.

**힐에코 시사점:** 웰니스 서비스라는 공통점이 가장 크며, "진행 중인 프로그램 데이터"를 보여주는 전략이 힐에코의 실천 기록 데이터와 직접 연결됩니다.

### 2-2. Calm (명상/수면)

**플로우 단계:**
1. 설정 > 구독 관리 > "구독 취소"
2. "취소하면 잃게 되는 것" 목록 — 수면 스토리, 가이드 명상, 일일 세션 등
3. 할인 오퍼 또는 플랜 변경 제안
4. 최종 확인
5. 해지 완료 — 현재 결제 주기 끝까지 이용 가능 안내

**핵심 전략:** 손실 회피 프레이밍. "얻을 수 있는 것"이 아니라 "잃게 되는 것"에 초점을 맞춥니다.

**힐에코 시사점:** 힐에코의 자가체크 결과, 수면 품질 추이, 실천 달력 등 축적된 데이터를 "잃게 되는 것" 프레이밍에 활용할 수 있습니다.

### 2-3. Noom (건강/체중 관리)

**플로우 단계:**
1. "구독 취소" 클릭
2. 해지 사유 선택
3. 리텐션 오퍼 — 월 $19 할인 플랜 제안 (65% 할인)
4. 일시 정지 옵션
5. 프리미엄 프로그램 업셀 제안 (해지를 업그레이드 기회로 전환)
6. 최종 확인
7. 해지 완료 — 남은 기간 이용 가능 안내 + 이메일 확인

**핵심 전략:** 다단계 리텐션. 할인 → 정지 → 업셀까지 3번의 이탈 방지 기회를 만듭니다. 다만 5단계 이상의 플로우는 "다크 패턴"으로 인식될 수 있다는 비판도 있습니다.

**힐에코 시사점:** 무료 체험 단계에서 할인 오퍼는 적용이 어렵지만, "일시 정지" 개념은 향후 유료 고객 해지에 활용 가능합니다.

### 2-4. Netflix (스트리밍)

**플로우 단계:**
1. "멤버십 해지" 클릭
2. "해지하면 놓치게 되는 콘텐츠" — 사용자 시청 이력 기반 추천 콘텐츠 표시
3. 플랜 다운그레이드 제안 (광고 포함 저가 플랜)
4. 최종 확인
5. 해지 완료 — 결제 주기 끝까지 시청 가능, 10개월 내 재가입 시 프로필 복원

**핵심 전략:** 개인화된 콘텐츠 추천으로 "이걸 보지 않고 떠날 건가요?" 감정을 유발합니다. Netflix는 이 전략으로 해지 의향 고객의 25%를 리텐션했습니다.

**힐에코 시사점:** 힐에코에서는 "아직 경험하지 못한 2주차 솔루션" 미리보기가 이 전략에 해당합니다.

### 2-5. Spotify

**플로우 단계:**
1. "Premium 해지" 클릭
2. 손실 회피 페이지 — "Premium 해지 시 잃게 되는 기능" 목록 (오프라인 저장, 광고 없는 재생 등)
3. 최종 확인
4. 해지 완료 — Free 계정으로 전환

**핵심 전략:** 단순한 손실 회피. 별도 할인이나 정지 옵션 없이, "잃게 되는 것" 목록만으로 리텐션을 시도합니다. 단순하지만, 중간에 맞춤 오퍼가 없어 전환율이 낮다는 분석도 있습니다.

### 2-6. YouTube Premium

**플로우 단계:**
1. "Premium 해지" 클릭
2. 개인화된 사용 지표 — "이번 달 광고 없이 시청한 시간: N시간", "저장한 영상: N개"
3. "해지 시 변경사항" 안내
4. 최종 확인

**핵심 전략:** 사용 데이터 기반 가치 증명. YouTube Premium은 개인화된 사용 지표를 보여줌으로써 해지율을 15% 감소시켰습니다.

**힐에코 시사점:** 힐에코의 실천일, 자가체크 변화, 수면 품질 변화 등 축적된 데이터가 이 전략에 직접 활용 가능합니다.

### 2-7. 한국 서비스 (밀리의서재, 왓챠)

**밀리의서재:**
1. 구독 관리 > 자동 결제 관리 > 해지 신청
2. 필수 동의 체크 + 해지 사유 선택
3. 해지 시 할인 혜택 쿠폰 제공 (리텐션 오퍼)
4. 해지 완료

**왓챠:**
1. 프로필 > 구독 정보 > "해지하기"
2. 안내 내용 확인
3. "해지할게요" 최종 확인
4. 해지 완료 — 구독 만료일까지 이용 가능

**공통 특징:** 한국 서비스들은 비교적 짧은 플로우(2~3단계)를 채택하며, 복잡한 리텐션 단계보다 "해지 후에도 남은 기간 이용 가능"을 강조하는 경향이 있습니다.

## 3. 업계 공통 패턴 요약

모든 사례에서 반복적으로 나타나는 해지 플로우의 4가지 핵심 요소:

| 요소 | 설명 | 적용 빈도 |
|---|---|---|
| 해지 사유 수집 | 다중 선택 설문 (4~6개 항목) | 거의 모든 서비스 |
| 손실 회피 프레이밍 | "잃게 되는 것" 강조 | Headspace, Calm, Spotify, YouTube |
| 맞춤 리텐션 오퍼 | 사유 기반 할인/정지/다운그레이드 | Headspace, Noom, Netflix |
| 정중한 마무리 | 남은 기간 이용 가능 + 재가입 안내 | 전체 |

## 4. 힐에코 free_trial 해지 플로우 설계안

### 4-1. 설계 원칙

1. **투명하고 간결하게** — 해지 과정은 최대 3단계 이내. 다크 패턴 없음.
2. **데이터로 가치 증명** — 힐에코가 보유한 실천 기록, 자가체크 변화, 수면 품질 변화를 활용.
3. **감정적 연결** — "당신이 쌓아온 것"에 초점. 압박이 아니라 공감.
4. **깨끗한 이별** — 해지 후에도 긍정적 브랜드 경험을 남겨 재가입 가능성을 열어둠.
5. **향후 앱 연동 고려** — iOS/Android 앱에서도 동일한 UX를 제공할 수 있는 구조.

### 4-2. 3단계 플로우

```
[Step 1] 해지 사유 선택
    ↓
[Step 2] 맞춤 리텐션 + "잃게 되는 것" 요약
    ↓
[Step 3] 최종 확인 → 해지 완료
```

### 4-3. Step 1 — 해지 사유 선택

**화면 구성:**
- 제목: "어떤 부분이 아쉬웠나요?"
- 부제: "솔직한 의견이 힐에코를 더 나은 서비스로 만듭니다."
- 사유 목록 (단일 선택):

| 코드 | 사유 텍스트 | 대상 유형 | 맞춤 리텐션 |
|---|---|---|---|
| `too_expensive` | "유료 전환 후 가격이 부담돼요" | B, C | 가격 안내 + 가치 대비 비용 |
| `no_time` | "바빠서 시간을 내기 어려워요" | B | "하루 15분" 재강조 |
| `not_effective` | "기대했던 효과를 느끼지 못했어요" | A, C | 변화 데이터 제시 |
| `found_alternative` | "다른 서비스를 이용하기로 했어요" | C | 힐에코 차별점 요약 |
| `just_trying` | "무료 체험만 해보려고 했어요" | D | 깨끗한 마무리 |
| `other` | "기타" (자유 입력) | 전체 | 일반 리텐션 |

### 4-4. Step 2 — 맞춤 리텐션 + "잃게 되는 것"

사유에 따라 상단 메시지가 달라지고, 하단에는 공통으로 "잃게 되는 것"을 개인화 데이터와 함께 표시합니다.

**사유별 상단 메시지 (예시):**

- `too_expensive`: "힐에코 월 구독료는 하루 커피 한 잔보다 적은 비용이에요. 주 3회 실천만으로 자율신경 밸런스 변화를 경험하실 수 있습니다."
- `no_time`: "하루 15분이면 충분해요. 출근 전, 잠들기 전, 가장 편한 시간에 시작할 수 있습니다."
- `not_effective`: (실천 데이터가 있으면) "{userName}님의 불균형 신호가 첫 체크 대비 {diff}%p 변화했어요. 꾸준한 실천이 더 큰 변화를 만듭니다." (없으면) "아직 솔루션을 경험하지 못하셨어요. 한 번만 시도해보시는 건 어떨까요?"
- `found_alternative`: "힐에코만의 특별한 점 — 피츠버그 수면 품질 분석(PSQI), 자율신경 밸런스 자가 체크, 주 단위 맞춤 솔루션 갱신을 다른 서비스에서도 경험하실 수 있나요?"
- `just_trying` / `other`: 상단 메시지 없이 바로 "잃게 되는 것"으로.

**하단 "잃게 되는 것" (개인화 데이터 기반):**

```
┌─────────────────────────────────────┐
│  해지하면 더 이상 이용할 수 없어요   │
│                                      │
│  📅 {practiceDays}일의 실천 기록      │
│  🌿 자율신경 밸런스 추이 데이터       │
│  🌙 수면 품질 분석 리포트             │
│  🧘 {currentWeek+1}주차 맞춤 솔루션   │
│  🎁 4주 실천 선물 영상                │
└─────────────────────────────────────┘
```

(실천 기록이 0일인 경우: "아직 경험하지 못한 솔루션이 기다리고 있어요"로 대체)

**CTA 버튼:**
- Primary: "체험 계속하기" (해지 철회)
- Secondary: "그래도 해지할게요" (Step 3으로)

### 4-5. Step 3 — 최종 확인 + 해지 완료

**해지 전 최종 확인:**
- "체험을 해지하면 {trialEndDate}까지 모든 콘텐츠를 이용할 수 있으며, 이후 자동 결제가 취소됩니다."
- 체크박스: "위 내용을 확인했습니다"
- CTA: "해지 완료" (POST /api/user/billing/cancel 호출)

**해지 완료 화면:**
- 아이콘: ✓
- "체험이 해지되었습니다"
- "{trialEndDate}까지 모든 콘텐츠를 이용하실 수 있어요."
- "언제든 다시 시작할 수 있습니다."
- CTA: "솔루션 페이지로 돌아가기" (남은 기간 이용 유도)

## 5. AWS 연동 사항

### 5-1. 기존 인프라 활용 (변경 불필요)

- `/api/user/billing/cancel` API Route: 프록시 이미 존재
- PaymentsTable: 빌링키 status → `cancelled` 로직 동작 중
- SubscriptionsTable: `free_trial → free_stopped` / `paid → paid_stopped` 전환 로직 동작 중
- **Step 2 사유별 리텐션 메시지**: 순수 프론트엔드 로직 — Lambda 변경 불필요
- **"잃게 되는 것" 데이터**: 이미 존재하는 API로 조회 가능 (practice-record, selfcheck-result, psqi-result, subscription)

### 5-2. 변경 필요 사항 — `billing-cancel` Lambda 수정

**현재 API 계약:**
```
POST /user/billing/cancel
Request:  { programId }
Response: { ok, message }
```

**변경 후 API 계약:**
```
POST /user/billing/cancel
Request:  { programId, cancelReason?, cancelFeedback? }
Response: { ok, message }
```

**변경 내용:**
- `billing-cancel` Lambda가 `cancelReason`(코드값)과 `cancelFeedback`(자유 입력 텍스트)을 선택적으로 수신
- SubscriptionsTable 업데이트 시 이 2개 필드를 함께 저장 (`cancelReason`, `cancelFeedback`, `cancelledAt`)
- 기존 로직(빌링키 취소 + 구독 상태 전환)은 변경 없음 — 필드 추가만

**수정 대상 파일:**
- `healecho-infra/lambda/billing-cancel.ts` — cancelReason, cancelFeedback 필드 저장 추가
- CDK 배포 필요: `cd healecho-infra && cdk deploy`

**이유:**
Step 1에서 해지 사유를, Step 2에서 자유 입력 피드백을 필수로 수집하므로, 이 데이터를 DynamoDB에 저장하지 않으면 수집 자체가 무의미합니다. 프론트엔드에서만 수집하고 저장하지 않는 것은 데이터 손실이며, 향후 이탈 분석과 winback 캠페인에 활용할 수 없습니다.

### 5-3. 결론

해지 플로우 구현에 필요한 작업:
1. **프론트엔드**: `/mypage/settings/subscription/cancel` 페이지 신규 생성 (Step 1~3) — 기존 API로 가능
2. **Lambda**: `billing-cancel.ts` 수정 — `cancelReason`, `cancelFeedback` 필드 저장 추가 (하위 호환 유지)
3. **CDK 배포**: Lambda 변경 후 `cd healecho-infra && cdk deploy`

## 6. 실제 설계

### Step 1 — 해지 사유 선택

**화면 제목:** "어떤 부분이 아쉬웠나요?"
**부제:** "솔직한 의견이 힐에코를 더 나은 서비스로 만듭니다."

**선택지 (단일 선택, 위에서 아래 순서):**

| 순서 | 코드 | 사유 텍스트 |
|---|---|---|
| 1 | `too_expensive` | 가격이 부담돼요 |
| 2 | `no_time` | 바빠서 시간을 내기 어려워요 |
| 3 | `not_effective` | 기대했던 효과를 느끼지 못했어요. |
| 4 | `found_alternative` | 다른 서비스를 이용하기로 했어요. |
| 5 | `just_trying` | 무료 체험만 해보려고 했어요. |
| 6 | `other` | 기타 (자유 입력) |

**UX 동작:**
- 1~5번은 라디오 버튼 형태로 하나만 선택 가능
- 6번 "기타" 선택 시 텍스트 입력 필드가 나타남
- 선택 완료 후 "다음" 버튼 → Step 2로 이동

### Step 2 — 사유별 맞춤 화면

**공통 골격:**

```
┌──────────────────────────────────┐
│  [상단] 사유별 맞춤 메시지        │
│  (1~2줄, 공감 또는 질문)         │
│                                   │
│  [중간] 사유별 콘텐츠             │
│  - 질문형: 자유 입력 필드         │
│  - 메시지형: 실천 데이터 카드      │
│  - 손실형: "잃게 되는 것" 목록    │
│                                   │
│  [하단] CTA 버튼 2개 (고정)       │
│  Primary + Secondary              │
└──────────────────────────────────┘
```

**사유별 상세 설계:**

#### 1. `too_expensive` — "가격이 부담돼요"

- **상단 메시지:** "고객의 의견을 적극 반영하겠습니다."
- **중간 콘텐츠:** 자유 입력 필드 (필수)
  - 질문: "힐에코에 어떤 서비스가 추가되면 가격 부담을 안 느끼실지 알려주세요.😊"
  - placeholder: `예: 1:1 맞춤 코칭, 식단 관리 기능, 오프라인 클래스 등`
- **CTA:**
  - Primary: "체험 계속하기"
  - Secondary: "그래도 해지할게요"

#### 2. `no_time` — "바빠서 시간을 내기 어려워요"

- **상단 메시지:** "하루 15분, 잠들기 전에 시작할 수 있어요."
- **중간 콘텐츠:** 메시지 카드
  - "작은 습관이 {userName}님의 1년을 바꿀 수 있습니다."
  - (실천 데이터가 있으면) "{userName}님은 지금까지 {practiceDays}일 실천하셨어요."
- **CTA:**
  - Primary: "체험 계속하기"
  - Secondary: "그래도 해지할게요"

#### 3. `not_effective` — "기대했던 효과를 느끼지 못했어요."

- **상단 메시지:** (실천 데이터가 있으면) "{userName}님은 지금까지 {practiceDays}일 실천하셨어요."
  - (실천 0일이면) "아직 솔루션을 경험하지 못하셨어요."
- **중간 콘텐츠:** 자유 입력 필드 (필수)
  - 질문: "힐에코에 어떤 서비스가 개선되면 더 좋은 효과를 느낄 수 있을지 알려주시면 적극 반영하겠습니다."
  - placeholder: `예: 더 다양한 난이도, 실시간 피드백, 체형별 맞춤 동작 등`
- **CTA:**
  - Primary: "체험 계속하기"
  - Secondary: "그래도 해지할게요"

#### 4. `found_alternative` — "다른 서비스를 이용하기로 했어요."

- **상단 메시지:** 없음 (질문이 바로 시작)
- **중간 콘텐츠:** 자유 입력 필드 (필수)
  - 질문: "다른 서비스의 어떤 면이 힐에코와의 차별점으로 느끼셨나요?"
  - placeholder: `예: 더 다양한 영상, 더 낮은 가격, 커뮤니티 기능 등`
- **CTA:**
  - Primary: "체험 계속하기"
  - Secondary: "그래도 해지할게요"

#### 5. `just_trying` — "무료 체험만 해보려고 했어요."

- **상단 메시지:** 없음
- **중간 콘텐츠:** "잃게 되는 것" 목록 (개인화 데이터 기반)

```
┌─────────────────────────────────────┐
│  해지하면 더 이상 이용할 수 없어요   │
│                                      │
│  📅 {practiceDays}일의 실천 기록      │
│  🌿 자율신경 밸런스 추이 데이터       │
│  🌙 수면 품질 분석 리포트             │
│  🧘 {currentWeek+1}주차 맞춤 솔루션   │
│  🎁 4주 실천 선물 영상                │
└─────────────────────────────────────┘
```

  - (실천 0일인 경우: "아직 경험하지 못한 솔루션이 기다리고 있어요."로 대체)

- **CTA:**
  - Primary: **"한 번만 더 경험해보기"** (다른 사유와 텍스트 차별화)
  - Secondary: "그래도 해지할게요"

#### 6. `other` — "기타"

- **상단 메시지:** "소중한 의견 감사합니다."
- **중간 콘텐츠:** Step 1에서 이미 자유 입력을 받았으므로 추가 입력 없음. "잃게 되는 것" 목록 표시 (5번과 동일).
- **CTA:**
  - Primary: "체험 계속하기"
  - Secondary: "그래도 해지할게요"

**공통 규칙:**
- 자유 입력 필드는 필수 입력이며, 최소 글자 수 제한 없음 ("없음", "모르겠음" 등도 허용)
- Secondary 버튼("그래도 해지할게요")은 회색 텍스트 버튼으로, 찾기 어렵지는 않되 시각적 위계를 낮춤
- "그래도 해지할게요" 클릭 시 → Step 3(최종 확인)으로 이동

### Step 3 — 최종 확인 + 해지 완료

**해지 전 최종 확인 화면:**

- 안내 문구: "체험을 해지하면 {trialEndDate}까지 모든 콘텐츠를 이용할 수 있으며, 이후 자동 결제가 취소됩니다."
- 체크박스: "위 내용을 확인했습니다" (필수 체크)
- CTA:
  - Primary: "체험 계속하기" (해지 철회 — 구독 관리 페이지로 복귀)
  - Danger: "해지 완료" (체크박스 체크 시에만 활성화 — `POST /api/user/billing/cancel` 호출)

**해지 완료 화면:**

- 아이콘: ✓
- 제목: "체험이 해지되었습니다"
- 안내: "{trialEndDate}까지 모든 콘텐츠를 이용하실 수 있어요."
- 부제: "언제든 다시 시작할 수 있습니다."
- CTA: "솔루션 페이지로 돌아가기" (남은 기간 이용 유도)
