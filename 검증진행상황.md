### 6. 보안 ✅

**발견된 이슈 (4건, 2건 수정 완료):**

| # | 이슈 | 심각도 | 수정 내용 |
|---|------|--------|----------|
| 1 | CORS `Access-Control-Allow-Origin: *` 와일드카드로 모든 외부 사이트에서 API 호출 가능 | **심각** | 환경변수 기반으로 변경 — prod: `NEXT_PUBLIC_SITE_URL`, dev: `http://localhost:3000` |
| 2 | 보안 헤더 없음 (X-Frame-Options, X-Content-Type-Options, HSTS 등) | 높음 | `next.config.js`에 5종 보안 헤더 추가 (HSTS는 prod만 적용) |

**미수정 (향후 별도 작업 권장):**
| # | 이슈 | 심각도 | 사유 |
|---|------|--------|------|
| 3 | 인증 토큰을 localStorage에 저장 (XSS 시 탈취 가능) | 중 | httpOnly 쿠키로 전환 필요 — 인증 전체 구조 변경이라 별도 작업 권장 |
| 4 | API Route(프록시)에서 JWT 서명 미검증 | 중 | upstream Lambda에서 Cognito 검증 수행 중이므로 이중 검증 여부는 설계 판단 |

**추가된 보안 헤더:**
- `X-Frame-Options: DENY` — 클릭재킹(다른 사이트에서 iframe으로 삽입) 차단
- `X-Content-Type-Options: nosniff` — 브라우저 MIME 타입 스니핑 차단
- `Referrer-Policy: strict-origin-when-cross-origin` — 외부 사이트로 URL 정보 유출 최소화
- `Permissions-Policy: camera=(), microphone=(), geolocation=()` — 불필요한 브라우저 API 차단
- `Strict-Transport-Security` (prod만) — HTTPS 강제 (2년)

**수정된 파일:**
- `next.config.js` — CORS 제한 + 보안 헤더 5종 추가
