# CLAUDE.md — Claude Code를 위한 프로젝트 지침서

## 사용자 정보
- 프로그래밍 경험이 전혀 없는 비개발자. AI 도움만으로 모든 것을 만들고 있음.
- 기술 용어는 항상 쉬운 비유나 일상적인 표현으로 설명할 것.
- 한국어(Korean)로 설명할 것. 코드와 코드 주석은 영어로 작성.

## 프로젝트: Heal Echo (힐에코)
다음을 제공하는 웰니스 솔루션:
- 요가 영상 콘텐츠
- 수면 습관 및 식습관 영상과 추적 기능
- '이해의 바다' 명상 세션
- 마음 습관 콘텐츠
- 자가진단 기반 정기 평가 및 맞춤형 웰니스 리포트

## 기술 스택
- 프론트엔드: Next.js 14 (App Router) + React 18 + TypeScript + Tailwind CSS
- 백엔드: AWS Lambda (TypeScript/Node.js) + API Gateway V2 (HTTP API)
- 데이터베이스: AWS DynamoDB (NoSQL)
- 인증: AWS Cognito (Google, Kakao, Naver, Apple SSO)
- 인프라: AWS CDK v2 (TypeScript) — S3, CloudFront, Lambda, EventBridge, AWS Backup, IAM
- 결제: Toss Payments SDK
- 주요 라이브러리: Axios, Zod, js-cookie

## 핵심 원칙
- 세계 최고 수준의 풀스택 개발자 및 AWS 아키텍트 수준으로 작업할 것.
- 코드를 보여주기 전에 항상 쉽고 단계적인 언어로 변경 사항을 설명할 것.
- 작업은 작은 단계로 나눌 것. 한 번에 너무 많은 변경을 하지 말 것.
- 이 웹 앱은 향후 모바일 앱과 통합될 예정. 모든 API는 웹 전용이 아닌 플랫폼에 종속되지 않는 방식으로 설계할 것.
- 인증은 JWT 기반으로 유지할 것. 세션 기반 로직을 절대 섞지 말 것.
- 파일 업로드는 항상 S3를 사용할 것. 서버 로컬에 파일을 절대 저장하지 말 것.
- 재사용성을 위해 컴포넌트를 분리할 것. 비즈니스 로직과 UI 로직을 명확하게 분리할 것.
- 하드코딩을 피할 것. 환경 변수와 설정 파일을 사용할 것.

## 개념 분리 (절대 섞지 말 것)
- **프로필 셋업 완료** = 온보딩 데이터 수집 (wellnessGoal, nickname, dietHabit 등). `profileSetupDone` 플래그로 판단.
- **솔루션 선택 확정** = 고객이 HOME에서 프로그램을 직접 선택. `program_confirmed` 플래그로 판단.
- 이 둘은 **완전히 독립된 개념**이다. 프로필 셋업의 wellnessGoal은 분석용 참조 데이터일 뿐, 솔루션 선택과 무관하다.
- 코드를 변경할 때 이 두 개념이 섞이고 있지 않은지 반드시 확인한다.

## 기존 사용자 영향 체크
- 인증, 프로필, 구독, 가드 로직을 수정할 때: 이미 AWS에 데이터가 있는 기존 사용자에 대한 영향을 반드시 확인할 것.
- 모든 사용자가 새 플로우를 거칠 것이라고 절대 가정하지 말 것. 기존 사용자는 몇 달 전에 이미 프로필을 완료했을 수 있고, 새 기기에서는 로컬 스토리지 데이터가 없을 수 있음.
- 새로운 가드나 리다이렉트를 추가하기 전: AWS 데이터(프로필, 환경설정, 구독)가 로컬 스토리지에 먼저 정상적으로 복원(hydrate)되었는지 확인할 것. 그래야 기존 사용자가 차단되지 않음.
- 두 가지 시나리오를 모두 테스트할 것: (1) 플로우를 처음 거치는 완전히 새로운 사용자, (2) AWS에 데이터가 있지만 로컬 스토리지가 비어있는 기존 사용자 (새 기기 / 캐시 삭제).
- 프로필 셋업의 wellnessGoal은 분석용 참조 데이터일 뿐이다. 사용자의 솔루션 선택을 결정하지 않는다. 솔루션 선택을 위한 고객 여정은 HOME 페이지에서 시작된다.

## 변경 안전 규칙
- **요청한 것만 변경**: 요청하지 않은 UI, 가드, 리다이렉트를 추가/삭제하지 않는다.
- **DB 데이터 확인 우선**: 코드만 읽고 추측하지 말고, 실제 DynamoDB에 어떤 필드가 존재하는지 확인한다.
- **되돌릴 수 없는 변경 금지**: `profileSetupDone=false`처럼 기존 데이터를 파괴적으로 변경하는 코드를 작성하지 않는다. 복원이 불가능한 변경은 반드시 사전 확인을 받는다.
- **텍스트 임의 수정 금지**: 기존 텍스트, 문구, 레이블, 안내 문구 등을 요청 없이 임의로 수정하지 않는다. 오타 수정이나 문구 변경이 필요해 보여도 반드시 사전 확인을 받는다.
- **가드 추가 전 체크리스트**: 새로운 가드/리다이렉트를 추가할 때 반드시 다음을 확인한다:
  1. 이 조건을 충족하지 못하는 기존 사용자가 있는가?
  2. AWS에 데이터가 있지만 로컬 스토리지에 없는 사용자는 어떻게 되는가?
  3. 기존 사용자가 이 가드에 의해 차단되면 어떤 일이 발생하는가?
