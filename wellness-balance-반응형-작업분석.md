# Wellness/Balance 태블릿 웹 & 모바일 웹 반응형 작업 분석서

**작성일:** 2026-03-24
**대상:** wellness/balance 페이지 + player 페이지 + 공통 컴포넌트
**목표:** 기존 웹(데스크탑) 디자인 유지 + 태블릿/모바일 웹 최적화 + 장기적 앱 연동 준비

---

## 1. 현재 상태 진단

### 1-1. 전체 아키텍처

| 항목 | 현재 상태 |
|------|-----------|
| 프레임워크 | Next.js 14 (App Router) + React 18 + TypeScript |
| 스타일링 | CSS Modules + Tailwind CSS v4 혼용 |
| 최대 너비 | `max-width: 720px` (모든 페이지 공통) |
| Viewport 설정 | `width=device-width, initialScale=1, userScalable=false, viewportFit=cover` |
| Safe Area | `env(safe-area-inset-*)` 적용됨 (Header, BottomTab) |

### 1-2. 현재 브레이크포인트

| 브레이크포인트 | 적용 범위 | 비고 |
|---------------|----------|------|
| `≤ 480px` | balance.module.css, Header | 모바일 소형 |
| `≤ 360px` | 일부 페이지 (profileSetup 등) | 초소형 기기 |
| **태블릿 구간 없음** | — | 768px~1024px 미대응 |

### 1-3. 파일별 현재 반응형 상태

| 파일 | 반응형 미디어쿼리 | 상태 |
|------|-----------------|------|
| `balance.module.css` | `@media (max-width: 480px)` 1개 | 태블릿 미대응 |
| `player.module.css` | **없음** | 반응형 전무 |
| `Header.module.css` | `@media (max-width: 480px)` 1개 | 태블릿 미대응 |
| `bottomTab.module.css` | **없음** | 반응형 전무 |
| `globals.css` | **없음** | 브레이크포인트 없음 |

### 1-4. 핵심 문제점

1. **태블릿 구간(768px~1024px) 완전 미대응** — 720px 초과 시 좌우 여백만 남음
2. **Player 페이지 반응형 부재** — 영상 플레이어가 기기별 최적화 안 됨
3. **BottomTab 반응형 부재** — 태블릿에서 탭 간격/크기 비최적화
4. **max-width: 720px 고정** — 태블릿에서 화면 활용도 낮음
5. **터치 인터랙션 미세 조정 부재** — hover 효과가 모바일에서 불필요하게 남아있음

## 2. 필요 작업 목록

### 2-1. 브레이크포인트 체계 수립 (신규)

현재 480px 하나뿐인 브레이크포인트를 다음과 같이 확장해야 합니다:

| 이름 | 범위 | 대상 기기 |
|------|------|----------|
| **Mobile S** | ≤ 360px | iPhone SE, Galaxy S 소형 |
| **Mobile** | ≤ 480px | iPhone 14/15, Galaxy S 표준 (기존) |
| **Mobile L** | ≤ 640px | iPhone Plus/Max, 대형 안드로이드 |
| **Tablet** | ≤ 1024px | iPad, Galaxy Tab, 태블릿 전반 |
| **Desktop** | > 1024px | 데스크탑 (현재 기준 유지) |

### 2-2. CSS 변경 작업 (파일별)

#### A. `globals.css` — 전역 반응형 기반 추가

- CSS 커스텀 속성(변수)에 브레이크포인트별 값 추가
- 컨테이너 최대 너비를 기기별로 분리 (720px → 태블릿 960px)
- 전역 터치 최적화 스타일 추가

#### B. `balance.module.css` — Balance 허브 페이지

| 작업 | 상세 |
|------|------|
| 태블릿 브레이크포인트 추가 | `@media (min-width: 641px) and (max-width: 1024px)` |
| 컨테이너 max-width 확장 | 태블릿: 960px, 모바일: 720px 유지 |
| 카드 레이아웃 변경 | 태블릿: 2컬럼 그리드 고려 / 모바일: 1컬럼 유지 |
| 이미지 aspect-ratio 조정 | 태블릿: `2.5/1` 또는 `3/1` / 모바일: `2/1` 유지 |
| 폰트 크기 단계 세분화 | `clamp()` 함수로 유동 타이포그래피 적용 |
| 패딩/간격 태블릿 최적화 | 태블릿에서 좌우 여백 확대 (24px~32px) |

#### C. `player.module.css` — 영상 플레이어 페이지 (반응형 전면 추가)

| 작업 | 상세 |
|------|------|
| 모바일 브레이크포인트 추가 | `@media (max-width: 480px)`, `@media (max-width: 360px)` |
| 태블릿 브레이크포인트 추가 | `@media (min-width: 641px) and (max-width: 1024px)` |
| 비디오 래퍼 반응형 | 모바일: 전체 너비 / 태블릿: 적정 여백 유지 |
| Paywall 모달 반응형 | 모바일: 하단 시트 형태 / 태블릿: 센터 모달 유지 |
| 비디오 컨트롤 터치 최적화 | 모바일에서 컨트롤 크기 확대 |

#### D. `Header.module.css` — 헤더 컴포넌트

| 작업 | 상세 |
|------|------|
| 태블릿 브레이크포인트 추가 | 로고 크기, 패딩 태블릿용 중간값 설정 |
| 가로 모드 대응 | `@media (orientation: landscape)` 추가 |

#### E. `bottomTab.module.css` — 하단 탭 네비게이션

| 작업 | 상세 |
|------|------|
| 태블릿 max-width 확장 | 720px → 960px (태블릿 시) |
| 아이콘 크기 반응형 | 태블릿: 32px / 모바일: 28px (현재) |
| 탭 라벨 추가 고려 | 태블릿에서는 아이콘+텍스트 라벨 표시 가능 |
| 가로 모드 대응 | landscape에서 사이드바 형태 전환 고려 |
| 모바일 터치 영역 확보 | 최소 48x48px 터치 타겟 보장 |

### 2-3. 컴포넌트(TSX) 검토 사항

코드 변경은 하지 않지만, 향후 필요한 컴포넌트 수준 작업을 미리 파악합니다:

| 컴포넌트 | 검토 항목 |
|----------|----------|
| `Header.tsx` | 태블릿에서 네비게이션 메뉴 확장 가능성 |
| `BottomTab.tsx` | 탭 라벨 텍스트 조건부 렌더링 (태블릿만) |
| `balance/page.tsx` | `Image` 컴포넌트 `sizes` prop 추가 (반응형 이미지 최적화) |
| `player/page.tsx` | 가로 모드 감지 및 전체화면 전환 로직 |

### 2-4. Next.js Image 최적화

현재 `Image` 컴포넌트에 `sizes` prop이 없어 모든 기기에서 동일 크기 이미지가 로드됩니다:

```tsx
// 현재 (비최적화)
<Image src="..." width={1536} height={1024} />

// 개선 필요
<Image src="..." width={1536} height={1024}
  sizes="(max-width: 480px) 100vw, (max-width: 1024px) 90vw, 720px" />
```

## 3. 장기적 앱 연동을 위한 준비 사항

### 3-1. 기술적 준비

| 항목 | 현재 | 권장 방향 |
|------|------|----------|
| PWA 설정 | 미적용 | `manifest.json` + Service Worker 추가 |
| 앱 셸 패턴 | 부분 적용 | Header + BottomTab 고정 레이아웃 강화 |
| 오프라인 지원 | 없음 | 핵심 페이지 캐싱 전략 수립 |
| 네이티브 제스처 | 미적용 | 스와이프 내비게이션, pull-to-refresh 고려 |
| 딥링크 | 미적용 | URL 스킴 설계 (`healecho://wellness/balance`) |

### 3-2. 앱 연동 전략 (단계별)

| 단계 | 전략 | 설명 |
|------|------|------|
| **1단계 (현재)** | 반응형 웹 완성 | 태블릿/모바일 웹 최적화 |
| **2단계** | PWA 전환 | 홈 화면 추가, 오프라인 지원, 푸시 알림 |
| **3단계** | 하이브리드 앱 | WebView 래핑 (Capacitor/React Native WebView) |
| **4단계** | 네이티브 전환 | React Native 또는 Flutter로 핵심 화면 네이티브화 |

### 3-3. 앱 전환 시 재사용 가능한 현재 자산

- API 레이어 (`/src/api`, `/src/config/constants.ts`) — 그대로 재사용
- 인증 로직 (`/src/auth`) — Cognito 기반이므로 네이티브에서도 동일 사용
- 비즈니스 로직 (`playerBrain.ts`, `canPlayVideo` 등) — 플랫폼 독립적
- 디자인 토큰 (CSS 변수) — 네이티브 스타일 시스템으로 매핑 가능
