# Balance 페이지 비즈니스 로직 재설계 명세서

> ⚠️ 이 문서는 비즈니스 전략 설계 명세입니다. 코드 변경 전 반드시 전체 내용을 숙지하세요.
> ⚠️ 기존 파일의 패턴을 먼저 확인하고 따라주세요.
> ⚠️ 한 번에 한 단계씩 진행하세요.

## 1. 고객 유형 정의 (3가지)

### 1-1. 둘러보는 고객 (Browser)
- 로그인은 했지만, 무료 체험 또는 유료 결제를 하지 않은 상태
- balance 페이지 진입 가능
- 달력 UI와 선물 표시를 볼 수 있음
- 영상 썸네일을 클릭하면 → 결제/무료체험 등록 화면으로 유도
- 영상 재생 불가

### 1-2. 무료 체험 고객 (Free Trial)
- 결제 정보를 입력한 후 무료 체험을 시작한 상태
- 무료 체험 기간: 프로그램 1주차 (시작일로부터 7일)
- 1주차 영상 시청 가능
- 1주차 종료 후 → 유료 결제가 자동으로 전환됨
  (결제 시스템 상세 설계는 추후 진행)
- 유료 전환 거부 시 → 영상 접근권 회수, 달력 기록은 유지
- 접근권 회수 후 재방문 시 → "여기까지 실천하셨어요. 이어서 해보시겠어요?" 메시지 표시

### 1-3. 유료 고객 (Paid)
- 활성 구독 상태
- 시작일 기준 7일마다 다음 주차 자동 해금
- 일시 정지 가능 → 복귀 시 고객에게 선택지 제공:
  - "이어서 계속하기" (중단한 주차부터 재개)
  - "처음부터 다시 시작하기" (1주차부터 재시작, 단 달력 기록은 보존)
- 일시 정지 중 시간은 프로그램 주차에 포함되지 않음 (주차 카운터 정지)

## 2. 프로그램 주차 체계

### 2-1. 주차 계산 방식
- 고객의 시작일 = 프로그램 1일차
- 프로그램 1주차 = 시작일 ~ 시작일+6일 (총 7일)
- 예: 수요일 시작 → 1주차는 수(1일)~화(7일), 2주차는 다음 수(1일)~화(7일)
- 달력 표시 시 프로그램 주차 구간을 시각적으로 구분

### 2-2. 주차 해금 조건
- 유료 고객: 시작일 기준 7일 경과 시 자동 해금
- 무료 체험 고객: 1주차만 해금, 2주차부터는 유료 전환 필요
- 둘러보는 고객: 모든 주차 잠금

### 2-3. 프로그램 특성
- Lifelong 프로그램 (총 주차 수 제한 없음)
- 프로그램별 독립 운영 (autobalance, womans-whisper, healing-mind 등)
- 현재 빌드 대상: autobalance만

---

## 3. 영상 시청 기록 시스템

### 3-1. 시청 인정 기준
- 일반 영상: 10분 이상 시청 시 "시청 완료"로 기록
- 선물 영상(30분): 시청 인정 기준 없음 (말 그대로 선물)
  - 선물 영상은 달력 시청 기록에 포함되지 않음
- 시청 시작 시각, 누적 시청 시간을 서버에 기록

### 3-2. 달력 시스템
- balance 페이지 내에 달력 UI 생성
- 영상을 시청한 날(10분 이상)에 시각적 표시
- 프로그램 주차 구간 구분 표시
- 모든 고객 유형에게 표시됨 (둘러보는 고객 포함)
- 접근권이 회수된 고객에게도 과거 기록은 유지하여 표시

### 3-3. 달력 데이터 보존 정책
- 무료 체험 만료 후: 기록 유지, 영상 재생만 차단
- 유료 구독 일시 정지 후: 기록 유지, 영상 재생 차단
- 복귀 시: 과거 기록을 보여주며 심리적 전환/복귀 유도
  → "여기까지 실천하셨어요. 이어서 해보시겠어요?"

## 4. 선물 시스템

### 4-1. 선물 획득 조건
- 프로그램 주차 기준 주 3일 이상 시청한 주간 = "달성 주간"
- 달성 주간 4회 누적 시 → 선물 영상 해금
- 반복형: 매 4회 달성마다 새로운 선물 영상 제공 (4주, 8주, 12주...)
- 프로그램별 독립 카운트 (autobalance 시청은 autobalance 선물에만 적용)

### 4-2. 선물 콘텐츠
- 30분짜리 동영상 프로그램
- 해금 후 1주일(7일) 동안만 시청 가능
- 1주일 경과 후 접근 불가
- 향후: 개별 구매 링크 삽입 예정 (현재 빌드하지 않음)
- 매 사이클마다 다른 내용의 선물 영상 제공

### 4-3. 선물이 제공되는 주차의 동작
- 선물 조건 달성 시, 해당 주차에는 일반 영상과 선물 영상 2개가 동시에 오픈
- 일반 영상: 기존 주차별 영상 (시청 추적 대상, 10분 이상 기준)
- 선물 영상: 30분짜리 보상 영상 (시청 추적 대상 아님, 기준 없음)
- 달력에는 일반 영상 시청만 기록됨

### 4-4. 선물 관련 메시지 시스템 (잔여 횟수 기반)

#### 첫 영상 시청 직후 (최초 1회):
"축하합니다! 이대로 계속 실천하시면 선물까지 4주 남았어요."

#### 진행 중 메시지:
- "4주 중 1주 달성! 선물까지 3주 남았어요"
- "4주 중 2주 달성! 선물까지 2주 남았어요"
- "4주 중 3주 달성! 선물까지 1주 남았어요"
- "축하합니다! 🎁 선물 영상이 도착했어요!"

#### 무료 체험 고객에게 (영상 시청 직후):
"주 3일 이상 4주 동안 실천하면 OOO님을 위한 선물이 준비되어 있어요!"

#### 둘러보는 고객에게:
- 달력 UI 내에 선물 마커를 표시 (시각적으로만)
- 별도 알림 메시지 없음

### 4-5. 미달성 시 처리
- 특정 주간에 3일 미만 시청 → 해당 주간은 달성 주간에 미포함
- 달성 카운터가 밀리는 것이지, 리셋되지는 않음
  - 예: 1주차 달성 → 2주차 미달성 → 3주차 달성 = "4주 중 2주 달성"
- 날짜가 아닌 잔여 횟수로 표현하여 실패감 최소화

## 5. Balance 페이지 UI 구조

### 5-1. 핵심 변경사항
- 현재주차에 해당하는 썸네일과 제목만 표시 (기존 3개 카드 → 1개)
- 선물 주차일 경우 일반 영상 + 선물 영상 2개 표시
- 하단에 달력 섹션 배치
- 선물 진행도 표시

### 5-2. 고객 유형별 동작

| 요소 | 둘러보는 고객 | 무료 체험 고객 | 유료 고객 |
|------|-------------|-------------|---------|
| 페이지 진입 | ✅ | ✅ | ✅ |
| 달력 표시 | ✅ (빈 달력) | ✅ (기록 포함) | ✅ (기록 포함) |
| 선물 마커 | ✅ (달력 내) | ✅ (달력 + 메시지) | ✅ (달력 + 메시지) |
| 현재 주차 썸네일 | ✅ (표시) | ✅ (1주차만 활성) | ✅ (해금된 주차) |
| 영상 클릭 | → 결제 유도 | → 재생 (1주차) | → 재생 |
| 잠긴 주차 클릭 | → 결제 유도 | → 결제 전환 유도 | → "X일 후 열립니다" |

## 6. 결제 시스템 (개요만 - 상세 설계 추후)

### 6-1. 무료 체험 진입 흐름
- 둘러보는 고객이 영상 클릭 시 → 결제 정보 입력 화면
- 결제 정보 입력 완료 → 무료 체험 시작 (프로그램 1주차)
- 무료 체험 기간(1주차) 종료 → 유료 결제 자동 전환

### 6-2. 현재 빌드 범위
- 고객 유형 판별 로직 (Browser / Free Trial / Paid)
- 영상 클릭 시 유형별 분기 처리
- 실제 결제 연동은 추후 설계

## 7. 데이터 모델 (필요한 신규 데이터)

### 7-1. 고객 구독 상태
- userId
- programId (예: "autobalance")
- subscriptionType: "browser" | "free_trial" | "paid"
- startDate (프로그램 시작일)
- currentWeek (현재 주차)
- status: "active" | "paused" | "expired" | "cancelled"
- pausedAt (일시 정지 시각, nullable)
- trialEndDate (무료 체험 종료일, nullable)

### 7-2. 영상 시청 기록
- userId
- programId
- weekNumber
- watchDate (시청 날짜)
- watchDurationSeconds (누적 시청 시간)
- isCompleted (10분 이상 여부)

### 7-3. 선물 사이클 진행도
- userId
- programId
- cycleNumber (1, 2, 3... 몇 번째 선물 사이클인지)
- qualifiedWeeks (달성 주간 수: 0~4)
- giftUnlockedAt (선물 해금 시각, nullable)
- giftExpiresAt (선물 만료 시각, nullable)
- giftVideoId (해당 사이클의 선물 영상 ID)

## 8. 작업 순서 (빌드 시)

| 순서 | 작업 | 설명 |
|------|------|------|
| 1 | 고객 유형 판별 로직 | subscriptionType 확인 함수 구현 |
| 2 | 영상 시청 추적 API | 시청 시간 기록/조회 엔드포인트 |
| 3 | 달력 컴포넌트 | 프로그램 주차 기반 달력 UI |
| 4 | 선물 시스템 로직 | 달성 주간 카운트, 선물 해금 판정 |
| 5 | Balance 목록 페이지 수정 | 고객 유형별 분기, 달력 삽입 |
| 6 | Balance 플레이어 수정 | 시청 추적, 완료 후 메시지 |
| 7 | 선물 영상 재생 페이지 | 7일 제한 접근 로직 |
| 8 | 일시 정지/복귀 UI | 선택지 화면 구현 |
| 9 | 통합 테스트 | 전체 플로우 검증 |

## 주의사항
1. 기존 파일의 패턴을 먼저 확인하고 동일한 방식으로 작성
2. 한 번에 한 단계씩 진행 (이전 단계 완료 후 다음 단계)
3. 각 단계 완료 후 npm run build 통과 확인
4. AWS 키는 절대 코드에 포함하지 않음 (환경 변수만 사용)
5. 코드 수정 시 전체 파일 교체 방식 사용 (부분 수정/diff 금지)
6. 결제 시스템 연동은 이 명세에 포함되지 않음 (추후 별도 설계)
