# Landing 페이지 검증 프롬프트 (새 작업에 복사하여 붙여넣기)

---

## 프롬프트 (아래 전체를 복사하세요)

나는 Heal Echo(힐에코) 웰니스 서비스의 public/landing 페이지를 검증하려 합니다.

### 대상 파일
- `src/app/public/landing/page.tsx` — 랜딩 페이지 메인
- `src/app/public/landing/IntroVideoClient.tsx` — 인트로 비디오 컴포넌트
- `src/app/public/landing/HeroProgramsClient.tsx` — 히어로 프로그램 컴포넌트
- `src/app/public/landing/landing.module.css` — 랜딩 페이지 스타일

### 검증 방법
'페이지별_코드검증_질문리스트.md'를 참조하여, 아래 순서대로 **한 카테고리씩** 검증하세요. 각 카테고리를 완료한 뒤 "다음 단계로 넘어갈까요?"라고 확인한 후 다음으로 진행하세요.

**1단계: 구조 & 코드 품질**
- 컴포넌트가 재사용 가능하게 분리되어 있는가?
- 상태 관리가 적절한가? props drilling이 과도하지 않은가?
- 하드코딩된 값이 있는가?

**2단계: 보안**
- XSS 방어가 되어 있는가?
- 민감 정보가 클라이언트 코드에 노출되지 않는가?
- 외부 리소스 로딩이 안전한가?

**3단계: API 설계**
- 이 페이지에서 호출하는 API 엔드포인트를 모두 찾고, RESTful 규칙과 에러 처리를 점검하세요.

**4단계: 에러 처리 & UX**
- 로딩/에러/빈 상태가 모두 처리되어 있는가?
- 비디오 로딩 실패 시 사용자에게 피드백이 있는가?

**5단계: 성능**
- 불필요한 리렌더링이 있는가?
- 이미지/비디오가 최적화되어 있는가? (lazy loading, 적절한 포맷)
- 번들에 불필요한 라이브러리가 포함되어 있지 않은가?

**6단계: 반응형 & 접근성**
- 모바일(375px), 태블릿(768px), 데스크톱(1280px)에서 레이아웃이 정상인가?
- 키보드 탐색, alt 텍스트, 색상 대비를 점검하세요.

**7단계: 앱 연동 대비**
- 향후 모바일 앱 연동 시 문제가 될 구조적 이슈가 있는가?

### 작업 규칙
- 위 대상 파일을 먼저 모두 읽은 뒤 검증을 시작하세요.
- 문제를 발견하면 심각도(높음/중간/낮음)를 표시하고, 수정된 코드를 함께 제시하세요.
- 설명은 한국어로, 코드와 주석은 영어로 작성하세요.
- 개발 지식이 없는 사람도 이해할 수 있도록 쉽게 설명하세요.
