# 나중에 체크하기 — Heal Echo 개선 작업 목록

> 이 문서는 코드 리뷰에서 발견된 개선 사항을 정리한 것입니다.
> AI에게 작업을 요청할 때 해당 섹션을 복사해서 프롬프트에 붙여넣으면 됩니다.
> 각 항목에는 대상 파일, 현재 상태, 해야 할 일이 명시되어 있습니다.

## 🔴 Major — 배포 전 수정 권장

### 1. AWS 저장 실패 시 사용자 피드백 없음

- **페이지**: profile-setup
- **대상 파일**: `profile-setup/page.tsx` (handleComplete 함수)
- **현재 상태**: 프로필 셋업 완료 시 AWS 저장이 실패해도 사용자에게 아무 알림이 없음
- **해야 할 일**: 저장 실패 시 토스트 메시지를 표시하거나, /home 진입 시 재시도 배너를 보여주기

### 2. route.ts 관리자 쿠키 폴백 제거 (Lambda 공개 API 구축 시 함께 처리)

- **페이지**: wellness/balance
- **대상 파일**: wellness/balance 관련 `route.ts`
- **현재 상태**: 공개 API인데 관리자 쿠키를 읽어서 인증 토큰으로 사용하는 임시 코드가 있음. 파일 주석에 "A단계 임시 해결"이라 명시되어 있음.
- **관련 코드**:
  ```typescript
  // 현재 코드 (제거 대상)
  if (!token) {
    const cookieName = process.env.ADMIN_AUTH_COOKIE || "heal_admin_auth";
    token = cookies().get(cookieName)?.value;
  }
  ```
- **해야 할 일**: Lambda 공개 API를 구축하면서 이 임시 쿠키 폴백 로직을 제거하고, 정상적인 인증 흐름으로 교체. 이 작업이 완료되면 아래 Structural Convergence 미해결 FAIL 3건(S-06, A-01, A-04)도 함께 해결됨.

## 🟡 Minor — 개선 권장 (급하지 않음)

### 3. 토큰 저장소를 localStorage에서 httpOnly 쿠키로 전환

- **페이지**: 전체 (인증 관련)
- **대상 파일**: 인증 토큰 저장/읽기를 담당하는 모든 파일
- **현재 상태**: JWT 토큰을 브라우저 localStorage에 저장 중. XSS 공격 시 토큰 탈취 가능.
- **해야 할 일**: 웹에서는 httpOnly 쿠키로 전환, 모바일 앱에서는 SecureStorage 사용. 웹/모바일 분기 설계가 필요하므로 계획을 세운 후 진행.
- **주의**: Heal Echo는 모바일 앱 연동 예정이므로, 쿠키 방식은 웹 전용으로 설계하고 모바일은 별도 토큰 저장 방식을 유지해야 함.

### 4. API 프록시에서 JWT 서명 검증 추가

- **페이지**: 전체 (API 프록시)
- **대상 파일**: Next.js `/api/` 경로의 프록시 라우트들
- **현재 상태**: 프론트엔드 API 프록시가 토큰의 진위 여부를 검증하지 않고 백엔드로 그대로 전달 중.
- **해야 할 일**: 프록시 단계에서 Cognito 공개키를 사용하여 JWT 서명 검증을 추가. 위조된 토큰을 백엔드 도달 전에 차단.

### 5. 네트워크 오프라인 감지 UI 추가

- **페이지**: login, wellness/balance (나아가 전체 앱)
- **현재 상태**: 네트워크가 끊겨도 사용자에게 별도 안내 UI가 없음.
- **해야 할 일**: `navigator.onLine`을 활용하여 오프라인 상태 감지 시 안내 UI 표시. 앱 전체에 공통 컴포넌트로 적용 권장.

### 6. SignupView Props 그룹화 (37개 → 객체 패턴)

- **페이지**: login
- **대상 파일**: `SignupView.tsx`
- **현재 상태**: SignupView에 전달되는 props가 37개. 기능은 정상이지만 유지보수 부담.
- **해야 할 일**: 관련 props를 그룹으로 묶는 객체 패턴 도입. 예: `passwordState`, `formState` 등으로 구조화.

### 7. PwRule 타입 중복 정의 통합

- **페이지**: login
- **대상 파일**: `SignupView.tsx`, `ForgotStep2View.tsx`
- **현재 상태**: PwRule 인터페이스가 두 파일에 각각 따로 정의되어 있음.
- **해야 할 일**: 공통 타입 파일(예: `types/auth.ts`)로 추출하여 한 곳에서 관리.

### 8. 소셜 로그인 객체 재생성 방지

- **페이지**: login
- **대상 파일**: `use-social-login.ts`
- **현재 상태**: `socialLoginUrlMap`과 `socialProviderLabel`이 훅 내부에서 매 렌더마다 재생성됨.
- **해야 할 일**: 해당 객체들을 모듈 스코프(파일 최상단)로 추출하여 불필요한 재생성 방지.

### 9. 보조 텍스트 색상 대비 개선

- **페이지**: profile-setup (나아가 전체 앱)
- **현재 상태**: 보조 텍스트 색상 `#9CA3AF`이 접근성 기준 대비가 부족함.
- **해야 할 일**: 앱 전체 디자인 시스템에서 `#6B7280` 이상으로 통일.

### 10. 성별 칩 터치 타겟 확대

- **페이지**: profile-setup
- **현재 상태**: 360px 화면에서 성별 선택 칩의 터치 타겟이 30px로 작음.
- **해야 할 일**: 모바일 접근성 기준인 44px 이상으로 확대.

### 11. localStorage PII 평문 저장 보안 개선

- **페이지**: profile-setup
- **현재 상태**: birthDate, gender 등 개인정보가 localStorage에 평문으로 저장됨.
- **해야 할 일**: 모바일 앱 전환 시 SecureStorage 검토. 웹에서도 민감 정보 암호화 저장 고려.

### 12. DenialOverlay 포커스 트랩 구현

- **페이지**: wellness/balance/player
- **대상 파일**: `player/page.tsx` (DenialOverlay 컴포넌트)
- **현재 상태**: `role="dialog"` 오버레이에 포커스 트랩이 없음. 키보드 사용자가 Tab으로 오버레이 뒤 콘텐츠에 접근 가능.
- **해야 할 일**: 오버레이 마운트 시 첫 번째 버튼에 autoFocus, Tab 키 순환을 오버레이 내부로 제한.

### 13. CDN Signed URL 적용

- **페이지**: wellness/balance
- **현재 상태**: 비디오 소스 URL이 클라이언트에 그대로 노출됨.
- **해야 할 일**: CloudFront Signed URL을 적용하여 비디오 URL 직접 접근 차단.

### 14. 에러 모니터링 도구 도입

- **페이지**: 전체 앱
- **현재 상태**: 에러 처리가 `console.error`와 `console.warn`에만 의존.
- **해야 할 일**: Sentry 등 구조화된 에러 리포팅 서비스 도입. 프로덕션 로그 정책 수립 후 console.warn도 일괄 정리.

### 15. 분석 추적(Analytics) 이벤트 추가

- **페이지**: login
- **현재 상태**: 로그인/회원가입/소셜 로그인 이벤트에 analytics 추적이 없음.
- **해야 할 일**: 주요 사용자 행동(로그인, 회원가입, 소셜 로그인)에 analytics 이벤트 추가.
